# 운영매뉴얼
**프로그램 포트폴리오 관리 시스템**

## 1. 시스템 관리

### 1.1 서버 모니터링
- **도구**: Prometheus + Grafana
- **확인 항목**:
  - CPU 사용률
  - 메모리 사용률
  - 디스크 사용률
  - 네트워크 트래픽
- **경보 기준**:
  - CPU > 80%
  - Memory > 85%
  - Disk > 90%

### 1.2 로그 관리
- **수집**: ELK Stack
- **보관**: 1년
- **검색**: Kibana

### 1.3 백업
- **전체 백업**: 매일 새벽 2시
- **증분 백업**: 매시간
- **복구 테스트**: 월 1회

## 2. 장애 대응

### 2.1 장애 등급
- **P1 (심각)**: 서비스 완전 중단
- **P2 (높음)**: 주요 기능 장애
- **P3 (중간)**: 일부 기능 장애
- **P4 (낮음)**: 사소한 이슈

### 2.2 대응 시간
| 등급 | 인지 | 해결 | 총 소요 시간 |
|------|------|------|-------------|
| P1 | 15분 | 4시간 | 4시간 15분 |
| P2 | 30분 | 1일 | 1일 30분 |
| P3 | 1시간 | 3일 | 3일 1시간 |
| P4 | 1일 | 7일 | 8일 |

### 2.3 장애 복구 절차
1. 장애 인지 및 등급 판정
2. 관련자 알림
3. 임시 조치 (완화)
4. 근본 원인 분석
5. 영구 조치 (해결)
6. 사후 보고서 작성

## 3. 정기 점검
- **일일**: 로그 확인, 성능 점검
- **주간**: 백업 확인, 보안 패치 점검
- **월간**: 장애 복구 훈련, 용량 계획
