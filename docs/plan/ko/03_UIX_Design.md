# UI/UX 설계 문서
**프로그램 포트폴리오 관리 시스템**

## 1. 디자인 원칙
### 1.1 핵심 원칙
- **단순성**: 3클릭 내 모든 기능 접근
- **일관성**: 전체 시스템 동일한 패턴
- **피드백**: 모든 액션에 즉각적 응답
- **접근성**: WCAG 2.1 Level AA 준수

### 1.2 디자인 시스템
**색상:**
- Primary: #6366F1 (Indigo)
- Success: #10B981 (Emerald)
- Warning: #F59E0B (Amber)
- Danger: #EF4444 (Red)

**타이포그래피:**
- H1: 32px, Bold
- H2: 24px, Bold
- Body: 14px, Regular

## 2. 주요 화면 설계
### 2.1 포트폴리오 대시보드
- 위쪽: KPI 카드 4개 (프로그램 수, 진행률, 예산, 리스크)
- 중간: 간트 차트, 리소스 히트맵
- 아래: 최근 활동, 예산 현황

### 2.2 리소스 관리
- 왼쪽: 팀원 리스트 (필터 포함)
- 중앙: 타임라인 뷰
- 오른쪽: 상세 정보 패널

### 2.3 프로젝트 상세
- 헤더: 프로젝트 정보
- 탭: 개요 / 작업 / 리소스 / 예산 / 리스크
- 메인: 선택된 탭 내용

## 3. 컴포넌트 라이브러리
### 3.1 버튼
- Primary (주요 액션)
- Secondary (보조 액션)
- Danger (삭제 등 위험 액션)
- Ghost (배경 없음)

### 3.2 폼
- 인라인 밸리데이션
- 에러 메시지 표시
- 필수 항목 표시 (*)

### 3.3 모달
- 크기: sm (400px), md (600px), lg (800px)
- 배경 클릭 시 닫기
- ESC 키로 닫기

## 4. 반응형 디자인
- Desktop: 1920px+
- Tablet: 768px - 1919px
- Mobile: < 768px
