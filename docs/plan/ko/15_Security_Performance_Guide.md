# 보안 및 성능 가이드
**프로그램 포트폴리오 관리 시스템**

## 1. 보안 베스트 프랙티스

### 1.1 인증
- JWT 토큰 만료: 15분
- Refresh 토큰: 7일
- MFA: 권장
- 비밀번호: 최소 12자, 영문+숫자+특수문자 조합

### 1.2 권한
- RBAC: 역할 기반 접근 제어
- 최소 권한: 필요한 최소한만 부여
- 정기 리뷰: 분기별 권한 재검토

### 1.3 데이터 보호
- 전송: TLS 1.3
- 저장: AES-256
- 민감 데이터: 추가 암호화

### 1.4 보안 패치
- OS: 월간 업데이트
- 라이브러리: 취약점 발견 시 즉시
- 자동 스캔: 일일

## 2. 성능 최적화

### 2.1 프론트엔드
- 코드 스플리팅: 라우트별 분할
- 이미지: WebP, lazy loading
- 번들 사이즈: 500KB 미만
- 캐싱: Service Worker

### 2.2 백엔드
- 데이터베이스 인덱싱
- 쿼리 최적화
- Redis 캐싱
- Read Replica

### 2.3 네트워크
- HTTP/2 사용
- 압축: Brotli
- CDN: 정적 리소스
- Prefetch: 다음 페이지

## 3. 모니터링

### 3.1 성능 메트릭
- TTFB: 100ms 미만
- FCP: 1.8s 미만
- LCP: 2.5s 미만
- CL: 0.1 미만

### 3.2 모니터링 도구
- Lighthouse: 정기 점검
- Web Vitals: 실제 사용자 데이터
- APM: 애플리케이션 성능

## 4. 문제 해결

### 4.1 성능 저하 원인
- N+1 쿼리
- 인덱스 누락
- 메모리 누수
- 큰 번들 사이즈

### 4.2 해결 방법
- 프로파일링: 병목 지점 파악
- 최적화: 인덱스 추가, 캐싱
- 테스트: 개선 확인
