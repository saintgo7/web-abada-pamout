# 배포 계획 문서
**프로그램 포트폴리오 관리 시스템**

## 1. CI/CD 파이프라인

### 1.1 CI (지속적 통합)
1. 코드 푸시
2. Lint 실행 (ESLint, Prettier)
3. 단위 테스트 (Jest)
4. 빌드 (Vite, tsc)
5. Docker 이미지 빌드
6. 이미지 푸시 (ECR/GCR)

### 1.2 CD (지속적 배포)
1. 개발 환경에 자동 배포 (모든 PR)
2. 스테이징 환경에 자동 배포 (main 브랜치)
3. 프로덕션 환경 수동 배포 (승인 후)

## 2. 환경 구성

### 2.1 개발 환경 (dev)
- 용도: 개발, 테스트
- 데이터: 샘플 데이터
- 접근: 내부 팀만

### 2.2 스테이징 (staging)
- 용도: 사전 테스트
- 데이터: 프로덕션 사본
- 접근: 내부 + 베타 고객

### 2.3 프로덕션 (prod)
- 용도: 실서비스
- 데이터: 실제 데이터
- 접근: 모든 사용자

## 3. 배포 전략
- **방식**: Blue-Green Deployment
- **롤백**: 이전 버전으로 즉시 복귀
- **다운타임**: 0 목표

## 4. 모니터링
- Prometheus + Grafana
- Sentry (에러 추적)
- CloudWatch (로그)
