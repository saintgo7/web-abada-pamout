# 코드리뷰 가이드
**프로그램 포트폴리오 관리 시스템**

## 1. 리뷰 목적
- 버그 조기 발견
- 코드 품질 향상
- 지식 공유
- 일관성 유지

## 2. 리뷰 프로세스
1. PR 생성
2. 자가 리뷰 (코드 작성자)
3. 리뷰어 지정
4. 리뷰 진행
5. 수정 요청 또는 승인
6. 병합 (Merge)

## 3. 리뷰 체크리스트

### 3.1 기능
- [ ] 요구사항 충족?
- [ ] 엣지 케이스 처리?
- [ ] 에러 처리?

### 3.2 코드 품질
- [ ] 명명 규칙 준수?
- [ ] 중복 최소화?
- [ ] 주석 적절?

### 3.3 성능
- [ ] 불필요한 루프 없음?
- [ ] N+1 쿼리 없음?
- [ ] 메모리 누수 없음?

### 3.4 보안
- [ ] 입력 밸리데이션?
- [ ] SQL 인젝션 방지?
- [ ] 인증/인가 확인?

## 4. 리뷰 원칙
- **존중**: 건설적인 피드백
- **구체적**: "여기가 이상해" 대신 "null 체크 필요"
- **설명**: 왜 수정이 필요한지 이유 제공

## 5. 승인 기준
- 모든 체크리스트 항목 통과
- 주요 이슈 없음
- 테스트 통과
- 문서화 완료
