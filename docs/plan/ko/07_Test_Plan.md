# 테스트 계획 문서
**프로그램 포트폴리오 관리 시스템**

## 1. 테스트 전략
- **단위 테스트**: 80% 커버리지 목표
- **통합 테스트**: API 엔드포인트 테스트
- **E2E 테스트**: 주요 사용자 시나리오
- **성능 테스트**: 부하 테스트
- **보안 테스트**: 취약점 스캔

## 2. 테스트 도구
- **단위**: Jest + React Testing Library
- **통합**: Supertest
- **E2E**: Playwright
- **성능**: k6
- **보안**: OWASP ZAP

## 3. 테스트 시나리오

### 3.1 기능 테스트
- 사용자 로그인/로그아웃
- 프로그램 생성/수정/삭제
- 리소스 할당
- 예산 추적
- 리포트 생성

### 3.2 성능 테스트
- 10,000 동시 사용자
- 5,000 RPS
- 응답 시간 < 200ms (p95)

## 4. 테스트 일정
- 단위 테스트: 매 커밋
- 통합 테스트: 매일
- E2E 테스트: 주 1회
- 성능 테스트: 월 1회
