## 1. 제품 개요

### 1.1 제품 정의

프로그램 포트폴리오 관리 시스템(PPMS)은 조직이 복수의 프로젝트와 프로그램을 효율적으로 관리할 수 있도록 지원하는 클라우드 기반 엔터프라이즈 소프트웨어입니다. 리소스 할당, 예산 추적, 일정 관리, 위험 관리, 이해관계자 커뮤니케이션 등 프로그램 관리의 전체 수명주기를 아우르는 종합적인 솔루션을 제공합니다.

### 1.2 문제 진단

**현재 조직이 직면한 문제:**
- 프로젝트 간 리소스 경합으로 인한 효율성 저하
- 실시간 프로그램 현황 파악의 어려움
- 수동 리소스 할당으로 인한 인적 오류 발생
- 예산 초과 사후 발견으로 인한 통제 불가
- 프로그램 간 의존성 관리의 부재
- 경영진을 위한 실시간 KPI 대시보드 부족

### 1.3 제품 솔루션

PPMS는 다음과 같은 해결책을 제공합니다:
- **중앙화된 포트폴리오 뷰**: 모든 프로그램과 프로젝트의 통합 관제
- **지능형 리소스 최적화**: AI 기반 리소스 추천 및 할당
- **실시간 예산 추적**: 예산 소진율 모니터링 및 경보
- **종합 위험 관리**: 위험 식별, 평가, 완화 계획 관리
- **자동화된 보고**: 사용자 정의 가능한 대시보드 및 리포트

### 1.4 타겟 시장

**1차 타겟:**
- 중견~대기업 (직원 100~5,000명)
- IT/소프트웨어 개발 조직
- 컨설팅 및 SI 기업
- 정부 및 공공기관

**2차 타겟:**
- 스타트업 (시리즈 B 이상)
- 교육 기관 (대학교, 연구소)
- 비영리 단체

## 2. 시장 분석

### 2.1 시장 규모

글로벌 프로젝트 포트폴리오 관리(PPM) 시장 규모:
- 2024년: 45억 달러
- 2029년 예상: 85억 달러
- 연평균 성장률(CAGR): 13.5%

### 2.2 경쟁 현황

**주요 경쟁사:**
1. **Atlassian (Jira, Portfolio for Jira)**
   - 강점: 강력한 이슈 추적, 널리 퍼진 생태계
   - 약점: 포트폴리오 관리 기능 약함, 복잡한 설정

2. **Microsoft (Project Online, Azure DevOps)**
   - 강점: Office 365 통합, 기업 안정성
   - 약점: 높은 비용, 낮은 사용자 경험

3. **Planview (Enterprise One)**
   - 강점: 전문 포트폴리오 관리, 강력한 리포팅
   - 약점: 매우 높은 비용, 긴 구현 기간

4. **Monday.com**
   - 강점: 직관적인 UI, 빠른 온보딩
   - 약점: 엔터프라이즈 기능 부족

### 2.3 우리의 경쟁우위

| 항목 | 경쟁사 대비 우위 |
|------|-----------------|
| **사용성** | 직관적인 UI/UX, 1시간 내 온보딩 |
| **AI 기능** | 지능형 리소스 추천, 위험 예측 |
| **가격** | 경쟁사 대비 40% 낮은 가격 |
| **통합** | 주요 도구(Slack, Teams, Jira)와 네이티브 통합 |
| **한국어 지원** | 네이티브 한국어 UI, 현지 지원 |

### 2.4 시장 진입 전략

**1단계 (0-6개월):** 국내 시장 점유
- 한국어 기능 최적화
- 로컬 기업 파일럿 프로젝트

**2단계 (6-18개월):** 아시아 태평양 확장
- 일본, 중국, 동남아 시장 진출
- 현지화 및 파트너십

**3단계 (18-36개월):** 글로벌 확장
- 북미, 유럽 시장 진출
- 엔터프라이즈 기능 강화

## 4. 제품 비전 및 목표

### 4.1 제품 비전

> "모든 조직이 데이터 기반으로 프로그램을 관리하고,
> 팀원들이 자신의 업무에 집중할 수 있는 세상을 만든다."

### 4.2 미션

1. **프로그램 관리의 민주화**: 누구나 쉽게 사용할 수 있는 도구 제공
2. **데이터 기반 의사결정**: 실시간 인사이트로 효율적인 의사결정 지원
3. **팀 워크플로우 자동화**: 반복 작업 자동화로 창의적 업무 집중 지원

### 4.3 핵심 목표 (2025년)

**사용자:**
- 유료 기업 고객 100개사 확보
- 월간 활성 사용자(MAU) 10,000명 달성
- 사용자 유지율 85% 이상 유지

**매출:**
- 연간 순매출(ARR) 20억 원 달성
- 고객당 평균 매출(ARPU) 월 200만 원
- 거래율 90% 이상

**제품:**
- 핵심 기능 100% 릴리스
- 플랫폼 가동율 99.9% 유지
- 사용자 만족도(NPS) 50점 이상

### 4.4 로드맵

**Q1 2025:** MVP 릴리스
- 프로그램/프로젝트 관리 기본 기능
- 리소스 관리
- 기본 보고

**Q2 2025:** 고급 기능
- 간트 차트
- 위험 관리
- 예산 관리

**Q3 2025:** 통합 및 자동화
- 타사 도구 통합 (Jira, Slack)
- AI 기반 리소스 추천
- 자동화된 리포팅

**Q4 2025:** 엔터프라이즈 기능
- SSO 기반 인증
- 고급 권한 관리
- 온프레미스 배포 옵션

## 5. 기능 요구사항

### 5.1 포트폴리오 대시보드 (Must Have)

**요구사항 ID:** REQ-DASH-001
**우선순위:** P0 (최우선)

**상세 설명:**
경영진과 프로그램 관리자가 모든 프로그램의 현황을 한눈에 파악할 수 있는 대시보드 제공

**핵심 기능:**
- 프로그램별 진행률 가시화 (원형/막대 차트)
- 리소스 활용률 현황 (히트맵)
- 예산 소진율 (게이지 차트)
- 리스크 레벨 표시 (신호등: 초록/노랑/빨강)
- 마일스톤 추적 (타임라인)
- KPI 메트릭 (표)

**필수 뷰:**
- 전체 포트폴리오 뷰
- 프로그램별 상세 뷰
- 개인 맞춤형 뷰 (드래그앤드롭 위젯)

**데이터 새로고침:**
- 자동: 5분마다
- 수동: 새로고침 버튼

### 5.2 리소스 관리 (Must Have)

**요구사항 ID:** REQ-RES-001
**우선순위:** P0

**상세 설명:**
프로그램 간 리소스(인원)를 효율적으로 할당하고 관리

**핵심 기능:**
- **리소스 풀 관리**
  - 팀원 등록 (이름, 역할, 스킬, 가용시간)
  - 스킬 태그 (Java, Python, Design, PM 등)
  - 가용 시간 설정 (주당 xx시간)

- **리소스 할당**
  - 프로젝트별 팀원 배정
  - 투입 비율 설정 (100%, 50%, 25% 등)
  - 기간 설정 (시작일~종료일)
  - 드래그앤드롭으로 간편 할당

- **리소스 로드맵**
  - 기간별 리소스 배치 현황 (타임라인 뷰)
  - 과부하 팀원 자동 표시 (빨간색 하이라이트)
  - 결원 리소스 추천

- **리소스 분석**
  - 활용률 리포트 (팀원별, 역할별)
  - 과부하/과소 사용 현황
  - 스킬 갭 분석

### 5.3 프로젝트/프로그램 관리 (Must Have)

**요구사항 ID:** REQ-PROJ-001
**우선순위:** P0

**상세 설명:**
프로젝트와 프로그램의 생명주기 관리

**핵심 기능:**
- **계층 구조**
  - 프로그램 > 프로젝트 > 작업(태스크)
  - 최대 5단계 깊이

- **프로젝트 생성**
  - 기본 정보: 이름, 설명, 담당자, 기간
  - 프로그램 연결
  - 예산 설정
  - 상태: 계획/진행중/완료/보류/취소

- **작업 관리**
  - 작업 생성 및 할당
  - 우선순위 (P0~P3)
  - 진행률 (0~100%)
  - 마일스톤 설정
  - 종속성 설정 (선후 관계)

- **일정 관리**
  - 간트 차트 뷰
  - 칸반 보드 (To Do, In Progress, Done)
  - 리스트 뷰
  - 캘린더 뷰

- **의존성 관리**
  - Finish-to-Start (FS): A 완료 후 B 시작
  - Start-to-Start (SS): A 시작 후 B 시작
  - Finish-to-Finish (FF): A 완료 후 B 완료
  - Start-to-Finish (SF): A 시작 후 B 완료

- **예산 경보**
  - 80%, 90%, 100% 도달 시 알림
  - 초과 시 승인 워크플로우

- **비용 리포트**
  - 프로그램별 비용 분석
  - 카테고리별 비용 분해
  - 추이 차트

**우선순위:** P1

  - 진행 상태

- **모니터링**
  - 정기 리뷰 주기 설정
  - 리스크 레벨 변경 알림

## 7. 사용자 스토리 및 유스케이스

### 7.1 경영진: 포트폴리오 현황 파악

**사용자 스토리:**
> 경영진으로서, 나는 전체 프로그램의 현황을 한눈에 파악하고 싶다.
> 그래야 전략적 의사결정을 내릴 수 있다.

**유스케이스:** UC-EXEC-001

1. 경영진이 시스템에 로그인한다
2. 포트폴리오 대시보드가 표시된다
3. 다음 정보를 확인한다:
   - 전체 프로그램 수 및 상태
   - 리소스 활용률
   - 예산 소진율
   - 리스크 레벨
   - 주요 KPI
4. 특정 프로그램을 클릭해 상세 내역을 확인한다
5. 필요한 경우, PDF 형식으로 리포트를 다운로드한다

**성공 조건:**
- 대시보드 로드 시간 2초 이내
- 모든 정보 실시간 (최대 5분 지연)
- 원클릭 리포트 다운로드

---

### 7.2 프로그램 관리자: 리소스 할당

**사용자 스토리:**
> 프로그램 관리자로서, 나는 제한된 리소스를 최적으로 할당하고 싶다.
> 그래야 모든 프로젝트를 기한 내에 완료할 수 있다.

**유스케이스:** UC-PM-001

1. 프로그램 관리자가 리소스 로드맵 뷰를 연다
2. 현재 리소스 배치 현황을 확인한다
3. 과부하 상태인 팀원이 표시된다 (빨간색)
4. 결원인 프로젝트를 확인한다
5. 팀원을 프로젝트로 드래그앤드롭해 할당한다
6. 시스템이 충돌을 경고한다 (이미 100% 할당된 경우)
7. 적절한 팀원을 찾아 재할당한다
8. 변경 사항을 저장한다

**성공 조건:**
- 과부하 자동 감지
- 충돌 시 실시간 경고
- 드래그앤드롭 직관적 동작

---

### 7.3 프로젝트 관리자: 일정 관리

**사용자 스토리:**
> 프로젝트 관리자로서, 나는 프로젝트 일정을 시각적으로 관리하고 싶다.
> 그래야 지연을 방지하고 마일스톤을 달성할 수 있다.

**유스케이스:** UC-PJM-001

1. 프로젝트 관리자가 간트 차트 뷰를 연다
2. 모든 작업과 마일스톤이 표시된다
3. 작업 기간을 드래그해 조정한다
4. 종속성이 있는 작업이 자동으로 날짜가 조정된다
5. 리소스를 할당한다
6. 진행률을 업데이트한다
7. 팀원에게 알림이 발송된다

**성공 조건:**
- 종속성 자동 계산
- 리소스 과부하 경고
- 실시간 협업 지원

---

### 7.4 팀원: 작업 관리

**사용자 스토리:**
> 팀원으로서, 나는 나에게 할당된 작업과 마감일을 명확히 알고 싶다.
> 그래야 업무를 체계적으로 처리할 수 있다.

**유스케이스:** UC-TM-001

1. 팀원이 시스템에 로그인한다
2. 내 작업 대시보드가 표시된다
3. 할당된 작업 리스트를 확인한다:
   - 작업명
   - 마감일
   - 우선순위
   - 진행률
4. 작업을 클릭해 상세 내용을 확인한다
5. 작업을 진행 중(In Progress)으로 변경한다
6. 실제 작업 시간을 기록한다
7. 완료하면 완료(Done)로 변경한다

**성공 조건:**
- 할당된 작업 명확히 표시
- 간단한 상태 변경
- 자동 시간 추적
