# Code Review Guide
**Program Portfolio Management System**

## 1. Purpose
- Early bug detection
- Code quality improvement
- Knowledge sharing
- Consistency maintenance

## 2. Process
1. Create PR
2. Self-review
3. Assign reviewers
4. Review
5. Request changes or approve
6. Merge

## 3. Checklist

### Functional
- [ ] Requirements met?
- [ ] Edge cases handled?
- [ ] Error handling?

### Code Quality
- [ ] Naming conventions?
- [ ] No duplication?
- [ ] Appropriate comments?

### Performance
- [ ] No unnecessary loops?
- [ ] No N+1 queries?
- [ ] No memory leaks?

### Security
- [ ] Input validation?
- [ ] SQL injection prevention?
- [ ] Auth/check?

## 4. Principles
- Be respectful
- Be specific
- Explain why
